## What We Do

AI is moving into every part of society. Our work is to make that shift more useful and more accountable: richer context, clearer ownership, better memory, stronger evaluation, and practical tools that help people build with AI instead of merely watching it happen.

<table>
  <tr>
    <td width="25%" valign="top">
      <h3>Products</h3>
      <p>We build Ubundi-born products such as tootoo and kwanda, each shaped around human context and practical AI adoption.</p>
    </td>
    <td width="25%" valign="top">
      <h3>Agent systems</h3>
      <p>We design agent workflows, memory layers, routing patterns, and operating systems for teams that need AI to do real work.</p>
    </td>
    <td width="25%" valign="top">
      <h3>Evaluation</h3>
      <p>We benchmark whether AI systems remember, reason, abstain, and stay grounded after the original context window is gone.</p>
    </td>
    <td width="25%" valign="top">
      <h3>Physical AI</h3>
      <p>Through First Motive, we are building ground-truth infrastructure for verified, multi-modal, train-ready robotics data.</p>
    </td>
  </tr>
</table>

## Open Work

Our public repositories are the parts of the studio we are building in the open: agent memory, context infrastructure, evaluation, developer guidance, and venture-facing experiments.

| Area | Repository | Why it matters |
| --- | --- | --- |
| Agent memory | [openclaw-cortex](https://github.com/Ubundi/openclaw-cortex) | Persistent memory for OpenClaw agents: auto-recall, auto-capture, tools, commands, file sync, and knowledge-graph retrieval. |
| Memory evaluation | [engram-benchmark](https://github.com/Ubundi/engram-benchmark) | Runtime benchmark for testing whether agents can recover grounded knowledge after the original context window is gone. |
| Context standards | [ubundi-agent-mode](https://github.com/Ubundi/ubundi-agent-mode) | A standard for making important artifacts readable by humans and usable by agents through claim maps, prompts, review gates, and freshness boundaries. |
| Identity context | [tootoo-openclaw](https://github.com/Ubundi/tootoo-openclaw) | An OpenClaw integration for bringing tootoo context into agent workflows. |
| Physical AI | [firstmotive-site](https://github.com/Ubundi/firstmotive-site) | First Motive's public surface for ground-truth infrastructure in Physical AI. |
| Experiments | [claw-journal](https://github.com/Ubundi/claw-journal) | Public experimental work around agent workflows and memory-adjacent tooling. |

## How We Think About AI

<table>
  <tr>
    <td width="33%" valign="top">
      <h3>Context first</h3>
      <p>Useful AI depends on the right source material, memory, and task framing. We build systems that know what to read before they act.</p>
    </td>
    <td width="33%" valign="top">
      <h3>Human-centred by default</h3>
      <p>Our products are designed around human identity, team trust, and real operating constraints, not demos that collapse outside the happy path.</p>
    </td>
    <td width="33%" valign="top">
      <h3>Compounding learning loops</h3>
      <p>We treat source quality, evaluation, review, and captured learning as core product infrastructure, because better context compounds.</p>
    </td>
  </tr>
</table>
